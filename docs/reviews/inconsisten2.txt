Internal inconsistencies and conceptual problems
Claims about GRU vs Mamba-2 contradict themselves:

Ablation table says “GRU backbone” gives lower world model loss (19.89) and higher return (137.7) than the full Mamba-2 model (193.7 baseline in Table 5, but later text also says “full DreamPrice model (return 124.3)” which is inconsistent).
​

Discussion alternately says GRU has “lower WM loss and slightly higher single-seed return than the full Mamba-2 model” and elsewhere that DreamPrice “exceeds all baselines” including model‑free RL.
​

The numbers for “Full DreamPrice return” are not consistent across sections (193.7 vs 124.3 vs normalized bar plot values).
​

Baseline comparison is internally incoherent:

In Table 4, DreamPrice has “Episode Return 194” while DQN has 4,291 and SAC −620 etc.
​

Later “Baseline comparison analysis” section refers to completely different numbers (e.g., Cost‑plus 42.1, Static XGBoost 87.2, DreamPrice 124.3, SAC 82.3), which do not match Table 4 at all and appear to be from a different experimental setup.
​

Units are mixed: sometimes “$/week”, sometimes “symlog reward”, sometimes unnamed; they are compared as if on the same scale.
​

Horizon/return numbers don’t line up:

Table 5: Full DreamPrice return 193.7, but horizon sweep text later uses 124.3 as “full model” reference, and Figure 5 normalizes around yet another value.
​

The horizon H=10 ablation is described both as “worst” and as having specific returns; but numbers used in different places are inconsistent (86.4 vs other normalized descriptions).
​

World‑model loss scales are inconsistent:

Table 5 lists WM loss values like 22.44 for the full model and 5.32 for deterministic latent (a 4× change), but earlier you describe ELBO components with magnitudes (e.g., reconstruction ~0.002, KL ~19.2, reward ~3.17) that do not obviously sum consistently to those scalar “WM loss” numbers as stated.
​

“No symlog+twohot” case: WM loss 573.2 is >25× the baseline; the narrative then still treats training as “stable” with similar training time, which is unlikely if the loss truly exploded that much.
​

Repeated “Limitations” sections:

You have a full “Limitations” paragraph in Section 4.6 and then another “Limitations” in Section 5, which partly repeat and partly diverge in details (e.g., seeds used, extent of evaluation).
​

They don’t agree on whether multi‑seed runs are executed vs “documented but not run”.

Incomplete or unsupported parts
No explicit experimental protocol for model‑free baselines:

DQN/PPO/SAC are said to be trained “within the world model” using SB3, but:

No clear statement of how many episodes/steps, replay buffer construction, behavioral policy, or reward scaling is given.
​

Yet you report precise episode returns and rank them.

Causal estimation section glosses over implementation:

Claims use DML‑PLIV with specific sample sizes and SEs, but there is no description of:

How controls X are chosen.

Which estimators (RF, GBM, nets) were used for nuisance functions.
​

How store/week fixed effects are handled.

Many figures are described but not justified:

Figures 3, 4, 5, 6, 7, 8 are narrative‑heavy, but:

Axes units are sometimes implicit (e.g., “World model loss” vs “Cumulative loss” vs “ELBO”), and may not correspond to the scalar losses in tables.
​

Some plots (OLS vs IV by store, bootstrap distributions) require substantial computation that you do not detail anywhere in Methods.

Replay sampling and environment setup:

“70% uniform across quarterly strata, 30% recent 2 years” replay sampler, and “GroceryPricingEnv wrapper” are referenced without formal definition or pseudocode.
​

The text claims this is “documented in the codebase”, but the paper itself is not self‑contained.

Questionable / overly strong framing
Very strong claims from single‑seed runs:

You repeatedly state or imply that DreamPrice “exceeds all baselines” or “dominates” alternatives, while also admitting that only a single seed is used “due to compute constraints.”
​

This is not statistically defensible; at best you can present it as an illustrative single‑run result.

Economic interpretation section might overreach:

You invoke Ramsey pricing and talk about “no finite monopoly optimum” and “competitive constraints bind prices well below monopoly levels” based only on a single estimated elasticity and log‑margin ranges, without actually computing any welfare or competitive equilibrium metrics.
​

Concrete edits I’d recommend
Tighten/soften novelty claims:

Change “first Dreamer-style world model for retail pricing/economic environments” to something like “we are not aware of prior work that…” and explicitly note the search scope.

Unify and audit all numeric tables/plots:

Pick a single, consistent set of numbers for:

Full DreamPrice return.

Baseline returns (Cost-plus, XGBoost, DQN, SAC, PPO).

Horizon ablations.

Ensure Table 4, Table 5, the horizon sweep, and the narrative all use the same metrics and units.

Remove or clearly mark placeholder URLs and hardware if they are not real:

Either verify that all GitHub / HF / Spaces / W&B links exist and correspond to the described artifacts, or mark them as “to be released” or remove them.

Replace “DGX Spark, GB10, CUDA 13.0” with the actual machine (e.g., A100, H100, etc.), or omit vendor‑specific naming entirely.

Tone down single‑seed claims:

Explicitly label all performance numbers as single‑seed illustrations and avoid words like “dominates”, “exceeds all baselines” unless you have multi‑seed IQM/CIs.

Clarify causal estimation:

Add a short subsection detailing:

Controls, fixed effects, and learners used in DML‑PLIV.

How the Hausman IV is constructed and checked (first‑stage regression form).

If bootstrap distributions, store‑level plots, etc. are based on synthetic or partial experiments, say so.

Either justify or remove detailed training‑time numbers:

If you did not actually measure per‑ablation wall‑clock times, remove Table 10 or replace with a qualitative statement about relative cost.
